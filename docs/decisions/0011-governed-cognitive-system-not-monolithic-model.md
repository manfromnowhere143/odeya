# 0011: Governed cognitive system, not a monolithic Odeya foundation model

- Status: Proposed
- Date: 2026-07-15
- Gate: G1/G2/G5/G6; requires cognitive-contract, matched-budget, scientific, security, and operator review

## Context

“Odeya engine” can be misread as a request to train one enormous proprietary model or build one immortal supervisor context. That choice would couple the company's scientific truth, state continuity, provider capability, safety, and improvement path to a probabilistic component whose outputs and behavior cannot own those responsibilities.

Current frontier evidence supports powerful model-driven search, hypothesis generation, tool use, and selective specialist coordination. It does not establish reliable general autonomous science, model self-verification, or scaffold-only repair of scientific reasoning failures. Odeya also needs to serve domains where deterministic computation, formal methods, calibrated instruments, expert judgment, physical testbeds, and human authority are stronger or mandatory oracles.

## Proposed decision

Odeya is a governed cognitive research system with two separate loops:

1. a deterministic scientific control plane owns canonical state, protocol, authority, rights, resources, evidence admissibility, verification requirements, recovery, and claim eligibility;
2. a replaceable model/tool fabric performs bounded epistemic search and returns proposals, artifacts, and graph deltas through exact contracts.

No model name, provider, prompt, context strategy, agent topology, memory database, or test-time search algorithm is constitutional. Each configuration is evaluated as a versioned whole under a resource vector, data/risk ceiling, validity window, and correlation profile.

Odeya may later train or fine-tune specialized models if retained evidence shows a precise capability, privacy, latency, cost, or scientific-validity need. Such a model remains an evaluated component behind the same contracts. Training a general Odeya foundation model is not a founding architecture dependency and would require a separate data-rights, compute, evaluation, safety, and business decision.

## Consequences

- Continuity survives model/context/provider replacement because state lives in events and artifacts.
- The strongest available model can be used for a role without granting it scientific or operational authority.
- Deterministic/statistical/physical oracles replace model judgment wherever they are stronger.
- A multi-agent coalition is admitted only where matched-budget evidence beats a single durable model baseline.
- Model debate/tournament scores guide exploration but cannot become evidence.
- Provider fallback creates a new attempt and cannot silently alter a confirmatory run.
- Cross-model diversity is one correlation axis, not proof of independence.
- Improvement remains eval-driven and quarantined; no component can rewrite its own production evaluator, policy, or authority.
- The company's defensible intellectual property centers on scientific state, evidence/claim semantics, the Research-State Compiler, verification, authority, recovery, evaluation, and domain adapters—not dependence on one vendor model.

## Rejected founding alternatives

### Train the “mother of all models” first

Rejected because it requires enormous data/compute before Odeya has proven the research task, lawful training corpus, scientific eval, or model-specific advantage. It also turns model limitations into system limitations and does not solve authority, evidence, physical validation, or publication.

### One immortal supervisor context

Rejected because context is finite, mutable, exposure-sensitive, difficult to audit, and not a reliable recovery store. Checkpointed canonical state and compiled role views are safer and replaceable.

### Unstructured autonomous swarm

Rejected because shared-state sequential work, correlated errors, cost, role confusion, and concurrent writers can worsen reliability. Topology follows task structure.

### Model consensus as the verifier

Rejected because correlated models can agree on the same error and lack world contact. Consensus ranks candidates; admitted evidence and independent verification settle claims.

### Production recursive self-modification

Rejected because a system that can alter its own objective, evaluator, policy, or evidence gate can optimize away the meaning of success. Improvement candidates remain outside production until independently evaluated and human-promoted.

## Falsifiers and review evidence

Keep this decision proposed if any of the following is true:

- the deterministic boundary cannot express the first-slice scientific distinctions without discretionary model judgment;
- the Research-State Compiler cannot reproduce role views or prevent sealed-truth influence;
- one model/harness baseline beats the proposed graph/coalition under matched budgets and equal verification;
- graph/specialist overhead reduces verified value or contradiction retention;
- models can route around command, authority, evidence, or publication admission;
- provider replacement changes mission/verdict semantics;
- the verification reserve can be exhausted by uncontrolled generation; or
- a proposed specialized training program lacks exact rights, contamination-safe evals, and a measurable advantage over replaceable frontier models.

Acceptance requires the cognitive contract schemas and traces, bounded control model, first-slice replay, matched-budget baseline plan, model/data-rights review, security review, and Daniel's signature on the exact architecture candidate. It would not select a provider or authorize runtime/model training.
